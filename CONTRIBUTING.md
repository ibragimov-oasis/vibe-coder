# 🤝 Contributing to Vibe-Coder Arsenal

Thank you for wanting to make this collection even better!

## What We're Looking For

### ✅ Yes, Please!

- **AI tools** that make coding agents smarter (skills, plugins, MCP servers)
- **Orchestration systems** that coordinate AI agents
- **Prompt libraries** and system prompt collections
- **UI/UX resources** that AI can use directly (component libraries, design systems)
- **Memory and context tools** for AI agents
- **Automation tools** that vibe-coders can use without writing code

### ❌ Not What We Need

- Tutorials, books, or "learn to code" repositories
- Repos that require you to understand code to use them
- Paid/closed-source tools
- Repos with fewer than 100 stars (exceptions for truly unique tools)

## How to Submit a New Project

1. **Fork** this repository
2. **Add the project** to the correct category folder (`Agents/`, `Orchestration/`, `Prompts/`, `Skills/`, `Tools/`, `UI-UX/`, or `Reference/`)
3. **Clean up** the project:
   - Remove promo videos and images  
   - Remove CONTRIBUTING.md (we use our own)
   - Keep README.md and CHANGELOG.md
   - Make hidden config files visible (rename `.claude/` → `_claude/` etc.)
4. **Update** the main `README.md` with the new project entry in the correct table
5. **Submit a Pull Request** with:
   - Project name and original repo link
   - One-line description
   - Why it's useful for vibe-coders
   - Star count at time of submission

## Code of Conduct

Be kind. We're all here because AI makes coding accessible. Let's keep it that way.
